# Наборы Хорошоп

Веб-сервис создает и управляет наборами товаров через API Хорошоп. Он сохраняет локальный реестр созданных наборов, потому что API Хорошоп не предоставляет массовую выгрузку уже существующих наборов.

## Запуск

1. В `config.json` укажите домен магазина. При первом запуске файл создается из `config.example.json`.
2. Установите зависимости: `pip install -r requirements.txt`.
3. Запустите `python sets_server.py`.
4. Откройте `http://<сервер>:8093`.

Логин и пароль API вводятся в веб-панели и не сохраняются. Вместо них можно указать действующий API-токен.

## Публичный лог

В `config.json` укажите папку, которая доступна для просмотра, и имя общего файла:

```json
"logging": {
  "public_log_path": "C:/ShareFiles/public",
  "public_log_name": "horoshop_sets.log"
}
```

В этот файл записываются сообщения контроллера обновлений, вывод Python-сервера, Uvicorn и необработанные ошибки. В JSON используйте прямые слеши `/` или двойные обратные `\\`; одиночные обратные слеши недопустимы. После изменения пути или имени перезапустите задачу `HoroshopSets`, чтобы сервис открыл новый файл. Если папка недоступна, сервис продолжит работу и временно использует локальный `logs\horoshop_sets.log`.

## Создание вручную

В форме нужны только три значения:

| Поле | Значение |
| --- | --- |
| Артикул набора | Необязательный уникальный артикул. Если оставить пустым, сервис составит его из артикулов отображения через точку. |
| Артикулы отображения товаров | Минимум два `article_for_display`, разделенные `;`. |
| Конечная цена | Цена набора после скидки. |

Сервис сопоставляет артикулы отображения с каталогом Хорошоп: сначала точное совпадение, затем совпадение без учета регистра. После успешного ответа набор записывается в локальный реестр.

Название каждого набора фиксировано: `Разом дешевше`. Валюта является серверной настройкой в `config.json`. В интерфейсе, Excel и API-запросах пользователь настраивает только конечную цену: процент скидки и порядок сортировки не передаются.

## Excel

Кнопка «Завантажити шаблон» отдает файл только с тремя колонками:

| Колонка | Значение |
| --- | --- |
| Артикул набору (необов'язково) | Артикул набора. Если оставить пустым, он будет составлен из артикулов товаров через точку. |
| Артикули відображення товарів | Не менее двух `article_for_display`, разделенных `;`. |
| Кінцева ціна | Конечная цена набора. |

В файле есть отдельный лист `Інструкція` с правилами заполнения. Колонки «Дія», «Активний», «Порядок сортування», «Знижка» и «Валюта» не используются.

Пустые дополнительные колонки игнорируются. Строка без отметки удаления создает или обновляет набор.

Кнопка «Вивантажити реєстр» добавляет четвертую колонку `Видалити (Так)`. Чтобы массово удалить отдельные наборы, в этой колонке напротив нужных строк впишите `Так`, затем загрузите файл и нажмите «Виконати операції». Пустая ячейка не означает удаление.

Удаление разрешено только для наборов, присутствующих в локальном реестре. Это защита от случайного удаления неизвестных наборов.

## Реестр и резервные копии

Текущие наборы хранятся в `data/sets_state.json`, история операций — в том же файле. Перед сохранением создается резервная копия в `data/backups`.

`config.json` и `data/sets_state.json` исключены из Git. Серверные скрипты перед `git reset --hard` резервируют и затем восстанавливают эти файлы, поэтому записи реестра не теряются при обновлении кода.

В таблице реестра можно изменить конечную цену, открыть полное редактирование состава, удалить один набор или выбрать несколько наборов для удаления. Таблицу также можно скачать и использовать для массового удаления.

## Windows Server

Запускайте PowerShell от имени администратора:

```powershell
.\FULL_SERVER_INSTALL.ps1
```

Установщик создает `config.json`, останавливает старую версию, выполняет `git reset --hard`, восстанавливает конфигурацию и реестр, проверяет зависимости и запускает фоновый контроллер. Контроллер проверяет Git каждые пять минут и при обновлении повторяет этот цикл без пересоздания виртуального окружения.

Перед автообновлением контроллер сохраняет `config.json` и реестр, затем проверяет новый код. Пакеты обновляются только при изменении `requirements.txt`. Если обновление или проверка не удалась, контроллер возвращает предыдущий коммит, восстанавливает данные и сам запускает прежнюю рабочую версию.

Для полной остановки сервиса:

```powershell
.\scripts\stop_tasks.ps1
```
