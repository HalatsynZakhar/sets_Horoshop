# Наборы Хорошоп

Веб-сервис создаёт и управляет наборами товаров через API Хорошопа. Локальный реестр хранит состав и параметры наборов, потому что API не предоставляет массовую выгрузку наборов.

## Настройка

1. Укажите домен магазина в `config.json`, например `https://myshop.horoshop.ua`.
2. Установите зависимости: `pip install -r requirements.txt`.
3. Запустите: `python sets_server.py`.
4. Откройте `http://<сервер>:8093`.

Логин и пароль API вводятся в веб-панели и не записываются в файлы. Вместо них можно использовать действующий API-токен.

## Реестр наборов

Все успешные создания и обновления сохраняются в `data/sets_state.json`. В записи есть артикул набора, реальные артикула товаров, артикула отображения, цена, название, активность, сортировка, скидка, валюта, ответ API и время операции.

`data/sets_state.json` является источником управления для сервиса. Он не удаляется при Git-обновлении; перед каждым сохранением также создаётся резервная копия в `data/backups`.

Важно: API Хорошопа из документации не умеет массово выгружать уже созданные наборы. Поэтому таблица реестра показывает только наборы, созданные через этот сервис или принятые на учёт. Удаление неизвестного реестру набора блокируется.

## Excel

Первые три колонки шаблона сохраняют простой формат:

| Колонка | Значение |
| --- | --- |
| Артикул набора | Уникальный текстовый артикул набора. |
| Артикулы отображения товаров | Не менее двух значений `article_for_display`, разделённых `;`. |
| Цена набора | Итоговая цена набора. |

Необязательные колонки: `Действие`, `Название`, `Активен`, `Порядок сортировки`, `Скидка %`, `Валюта`.

Допустимые действия:

- `обновить`: создать или обновить набор в Хорошопе;
- `удалить`: удалить только набор, уже находящийся в локальном реестре;
- `принять на учет`: сохранить набор в реестре без вызова API Хорошопа.

Реестр можно выгрузить в Excel, массово поменять параметры или отметить строки для удаления и загрузить обратно.

## Поддерживаемые параметры

Для импорта используются актуальные поля API: `article`, `title`, `discountedPrice`, `currency`, `enabled`, `sortOrder`, `discountPercent`, `products`. Устаревшее `initialPrice` не отправляется.

Переименование артикула набора требует создать новый набор и удалить старый: API не предоставляет отдельной операции переименования.

## Windows Server

Запускайте PowerShell от имени администратора:

```powershell
.\FULL_SERVER_INSTALL.ps1
```

Установщик создаёт `config.json` из шаблона, останавливает старую версию, выполняет `git reset --hard`, устанавливает зависимости, открывает Firewall, создаёт бессрочную задачу `HoroshopSets` и проверяет запуск веб-сервера. `config.json` и `data/sets_state.json` отдельно резервируются и восстанавливаются после обновления кода.

`scripts\supervisor.ps1` проверяет Git каждые пять минут. При обновлении он останавливает веб-сервер, выполняет `git reset --hard`, обновляет зависимости в существующей `.venv`, проверяет Python-файлы и запускает новую версию.

Полная остановка сервиса:

```powershell
.\scripts\stop_tasks.ps1
```
