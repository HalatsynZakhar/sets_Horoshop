# Набори Хорошоп

Веб-сервіс створює та оновлює комплекти товарів через API Хорошопа.

## Налаштування

1. Вкажіть домен магазину в `config.json`, наприклад `https://myshop.horoshop.ua`.
2. Встановіть залежності: `pip install -r requirements.txt`.
3. Запустіть: `python sets_server.py`.
4. Відкрийте `http://<сервер>:8093`.

Логін і пароль API вводяться у веб-панелі та не записуються у файл. За потреби можна вказати чинний API-токен замість логіна і пароля.

## Excel

Скачайте шаблон у панелі. У ньому використовуються три стовпці:

| Стовпець | Значення |
| --- | --- |
| Артикул набору | Унікальний артикул, який не повинен збігатися з артикулом товару. Вводьте як текст, щоб Excel не прибрав початкові нулі. |
| Артикули товарів | Щонайменше два текстові значення «Артикул відображення на сайті», розділені `;`. |
| Ціна набору | Підсумкова ціна набору. |

Перед імпортом сервіс вивантажує каталог, звіряє компоненти за `article_for_display` (спочатку точний збіг, потім без урахування регістру) і показує помилки. До API відправляються реальні артикули товарів.

Після кожного імпорту в `data/sets_state.json` зберігаються артикул набору, склад, ціна, статус, повідомлення API та час оновлення. Стан є локальним журналом сервісу; він не замінює дані Хорошопа.

## Розгортання на Windows Server

Скрипти для Windows підготовлені за схемою попереднього проєкту:

- `FULL_SERVER_INSTALL.ps1` клонує або оновлює репозиторій, створює `.venv`, встановлює залежності, відкриває порт у Firewall, створює безстрокове завдання Планувальника `HoroshopSets` і перевіряє запуск веб-сервера. Хід установки записується до `C:\ProgramData\HoroshopSets-install.log`.
- `scripts\supervisor.ps1` тримає веб-сервер запущеним і перевіряє оновлення Git кожні п'ять хвилин. Під час оновлення він примусово приводить код до `origin/main` через `git reset --hard`.
- `scripts\stop_tasks.ps1` зупиняє сервер. За замовчуванням він також видаляє завдання Планувальника; параметр `-KeepTask` лише зупиняє процеси.

Запустіть PowerShell від імені адміністратора та виконайте:

```powershell
.\FULL_SERVER_INSTALL.ps1
```

Під час інсталяції відкриється `config.json`: вкажіть реальний `horoshop.domain`. `config.json`, `data`, журнали та віртуальне середовище не потрапляють до Git, тому примусове оновлення коду їх не перезаписує. Усі неігноровані локальні зміни в каталозі проєкту буде видалено.

Якщо `config.json` ще немає, інсталятор створює його з `config.example.json`. Під час кожного повторного запуску наявний `config.json` окремо резервується й відновлюється після оновлення коду. Веб-сервіс читає цей файл при запуску.
